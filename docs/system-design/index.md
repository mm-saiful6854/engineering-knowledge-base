# System Design

System design is the practice of shaping software so it can handle real users, real failures, changing requirements, and operational constraints.

This section starts with foundations, then moves into protocols, API styles, architecture patterns, and modeling concepts.

## How To Practice

- Pick one concept and explain it in plain language.
- Draw a small diagram from memory.
- Identify when the concept is useful and when it is overkill.
- Compare it with at least one alternative.
