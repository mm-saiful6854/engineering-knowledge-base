# Contributing

Thanks for helping improve this engineering knowledge base.

## Good Contributions

- Fix incorrect technical explanations.
- Improve clarity, examples, diagrams, and references.
- Add concise notes for missing concepts.
- Report broken links, unclear wording, or outdated references.

## Writing Style

- Write in your own words.
- Prefer short explanations with concrete examples.
- Keep each article useful for revision: summary, why it matters, core concepts, common mistakes, related topics, and references.
- Put source links under `References` or `Further Reading`.
- Do not paste large copied sections from books, courses, articles, or documentation.

## Pull Request Checklist

- The page has a clear title and short summary.
- Internal links use clean relative Markdown paths.
- Images or diagrams have meaningful names and alt text.
- References are included for non-obvious facts or external learning material.
- `mkdocs build --strict` passes locally when possible.

## Issues

Use issues for corrections, topic requests, missing references, or questions about how a note should be organized.
